# Contribution Scorecard
How technical contributions are evaluated (objectively, no social bias):

## Code Contributions
- Security audit passes: 10 points
- Implements spec exactly: 8 points  
- Includes tests: 5 points
- Clean documentation: 3 points

## Documentation Contributions
- Clarifies complex topic: 5 points
- Fixes technical error: 3 points
- Adds examples: 2 points

## Points → Grant Eligibility
- 50+ points: Eligible for small grants ($500-2000)
- 100+ points: Eligible for feature grants ($2000-10000)
- All points publicly visible in contributor file

## How Points Are Tracked
1. Contributor submits PR
2. Automated checks run (tests, security scans)
3. Code review against specification
4. Points awarded based on objective criteria above
5. Points logged in CONTRIBUTORS.md file

## Tracking & Transparency
All points are tracked in a public CONTRIBUTORS.md file with:
- Contributor name/GitHub handle
- PR numbers and descriptions
- Points awarded
- Cumulative total

This creates an objective, merit-based path to compensation through grants.
